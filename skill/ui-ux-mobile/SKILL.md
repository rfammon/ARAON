## Core UX/UI Principles (REQUIRED)

Siga estes princípios fundamentais para criar experiências móveis excepcionais.

### 1. User-Centered Design (Design Centrado no Usuário)
*   **Empatia:** Entenda as dores e necessidades do usuário. Crie personas.
*   **Objetivos Claros:** O design deve ajudar o usuário a completar tarefas com o mínimo de atrito.
*   **Feedback:** Forneça feedback visual imediato para todas as interações (cliques, carregamento, erros).

### 2. Simplicidade e Clareza (Minimalismo)
*   **Menos é Mais:** Evite desordem visual. Foque no conteúdo essencial.
*   **Hierarquia Visual:** Use tamanho, cor e posição para guiar o olhar do usuário para o que importa.
*   **Reduza Input:** Minimize a digitação. Use autopreenchimento, seletores e logins sociais.

### 3. Consistência
*   **Design System:** Mantenha cores, tipografia, ícones e espaçamentos consistentes em todo o app.
*   **Padrões de Plataforma:** Respeite as diretrizes nativas (iOS Human Interface Guidelines / Material Design 3) para familiaridade.

## Mobile Specifics

### Ergonomia e Toque
*   **Touch Targets:** Botões devem ter pelo menos **44x44pt (iOS)** ou **48x48dp (Android)**.
*   **Thumb Zone:** Coloque ações principais na parte inferior da tela para facilitar o uso com uma mão.
*   **Legibilidade:** Tamanho de fonte base mínimo de **16sp/pt** para texto corrido. Contraste adequado (WCAG AA).

### Navegação
*   **Barra de Abas (Bottom Navigation):** Para navegação principal (3-5 destinos).
*   **Hambúrguer:** Apenas para destinos secundários.
*   **Voltar:** Sempre forneça uma maneira clara de voltar (além do gesto nativo).

## Processo Recomendado

1.  **Discovery:** Pesquisa, Personas, User Stories.
2.  **Wireframing:** Estrutura de baixa fidelidade (papel ou digital) para validar fluxos.
3.  **Prototyping:** Alta fidelidade com interações para testar a experiência.
4.  **Testing:** Testes de usabilidade com usuários reais antes do desenvolvimento.
5.  **Handoff:** Preparação de assets e especificações para desenvolvimento.
